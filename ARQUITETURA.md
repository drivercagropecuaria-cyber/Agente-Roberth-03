## Stack TГ©cnica

### Backend
| Componente | Tecnologia | Justificativa |
|-----------|-----------|---------------|
| Runtime | Node.js (TypeScript) | Ecossistema rico, OpenAI SDK oficial |
| Framework | Fastify | Performance, schema nativo, plugins |
| Agentes | OpenAI Agents SDK (JS) | Handoffs, tracing, guardrails nativos |
| Filas | Supabase pgmq | Durabilidade, sem infra extra |
| Jobs worker | Worker separado | Desacoplamento, resiliГӘncia |

### Frontend
| Componente | Tecnologia | Justificativa |
|-----------|-----------|---------------|
| Framework | React + Vite + TypeScript | PadrГЈo moderno, performance |
| Estilo | Tailwind CSS | Utilidades, consistГӘncia visual |
| UI Components | shadcn/ui | Componentes acessГӯveis e customizГЎveis |
| Estado | React Query + Zustand | Cache, estado global simples |
| Realtime | Supabase Realtime | Updates ao vivo no dashboard |
| Roteamento | React Router v6 | SPA com mГәltiplas rotas |

### Banco de Dados
| Componente | Tecnologia | Justificativa |
|-----------|-----------|---------------|
| Principal | Supabase (PostgreSQL) | Dados estruturados, RLS, Auth |
| Storage | Supabase Storage | Artifacts, apresentaГ§Гөes HTML |
| Filas | pgmq (via Supabase) | Jobs durГЎveis, retry nativo |
| Realtime | Supabase Realtime | Updates em tempo real |

### IntegraГ§Гөes
| IntegraГ§ГЈo | Uso |
|-----------|-----|
